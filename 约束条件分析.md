## 四、安全约束

### 4.1 急停机制（最高优先级）

| 约束 | 说明 |
|------|------|
| 急停不依赖身份认证 | 任意人员均可触发 |
| 语音"停止/急停"绕过声纹门禁 | 最高优先级，无视认证状态 |
| 物理急停按钮独立安全通道 | 与软件解耦 |
| 火灾告警 > 所有业务权限 | 有声纹认证也不能忽略火情 |
| 高危火情只能"确认/接管/返航" | 普通语音命令被阻止 |
| ⚠ **急停后自动恢复策略未定义** | 当前只定义需人工确认 |

### 4.2 串口安全

| 约束 | 说明 |
|------|------|
| `_move_lock` 互斥锁 | 防多线程同时写串口致 MCU 死机 |
| `_stop_timer` 定时器替换 | 防多线程抢串口 |
| api_server.py 与 app.py 二选一 | 防串口争用 |

### 4.3 数据安全

| 约束 | 说明 |
|------|------|
| 车端不直连 MySQL | 所有写库经接收 API，库凭据不下发 |
| Bearer Token 鉴权 | `/healthz` 除外 |
| API Key 禁止提交 Git | TTS/ASR/LLM Key 均不可 commit |
| 车牌/人脸仅存必要信息 | 演示后清理原始样本 |
| 连续认证失败锁定 | 记录安全事件 + APP/管理员解锁 |
| 上传大小限制 | 云端 `MAX_UPLOAD_MB=10` |

### 4.4 告警安全

| 约束 | 说明 |
|------|------|
| 幂等去重 | `(event_id, alarm_type)` 唯一索引 |
| 断网离线队列 | `outbox.jsonl` + 指数退避重试 |
| ⚠ 告警上报失败但状态已标记 ALARMED | **已知 Critical 风险** |
| ⚠ 碰撞检测只告警不制动 | **已知 Major 风险** |

## 八、缺失约束清单

以下是当前代码和文档中应存在但缺失的约束条件，分三个优先级：

### 🔴 Critical — 可能造成安全事故或核心功能永久失效

| # | 缺失的约束 | 风险 |
|---|-----------|------|
| 1 | **电池电压下限截止阈值**（只读取不保护 → 过放） | 永久损坏电池 |
| 2 | **电机最大速度硬限位**（仅 `min(speed,100)` 软件截断可绕过） | 底盘失控 |
| 3 | **电机堵转/过流保护** | 电机烧毁 |
| 4 | **碰撞检测自动制动**（当前只告警不制动） | 碰撞事故 |
| 5 | **YOLO 子进程存活监控 + 自动重启** | 火灾漏检 |

### 🟠 Major — 影响系统可靠性或可用性

| # | 缺失的约束 | 风险 |
|---|-----------|------|
| 6 | **传感器读取线程心跳 + 看门狗** | 环境数据过期 |
| 7 | **AI Reviewer worker 心跳 + 自动重建** | AI 复核全线不可用 |
| 8 | **急停后的恢复授权流程** | 操作歧义 |
| 9 | **导航失败恢复策略**（定位丢失后行为未定义） | 迷航失控 |
| 10 | **WiFi 断开后的小车安全行为**（继续巡逻？停车？） | 远程不可控 |
| 11 | **温度/湿度超限告警阈值**（有传感器但无规则） | 环境风险未检测 |
| 12 | **连续认证失败锁定策略**（次数、冷却期未定义） | 暴力攻击面 |
| 13 | **磁盘空间使用率监控**（证据+日志持续写入） | 磁盘满 crash |

### 🟡 Minor — 影响运维质量或验收标准

| # | 缺失的约束 | 风险 |
|---|-----------|------|
| 14 | **端到端时延 SLA 承诺**（500ms/3s 是 PPT 目标，非实测） | 验收标准不明 |
| 15 | **底盘超时停止机制的可靠性**（Timer 线程调度不可靠） | 失控持续 |
