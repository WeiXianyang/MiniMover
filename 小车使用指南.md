## 1. 系统概览

### 1.1 产品简介

iCar 智能小车是一个基于 OpenHarmony 操作系统的多场景巡检实训开发平台，涵盖物联网、自动化、机器人、人工智能、自动驾驶等多学科。同时运行通用机器人 ROS 2 系统以及基于国产鸿蒙 AT32 控制板的底盘麦轮运动控制系统。

车上搭载了深度相机、激光雷达、气压传感器、温湿度传感器、NBIOT 等多种环境感知传感器模块，可以进行多学科及跨学科的开发及实验研究。环境感知模块采用磁吸设计，拆装灵活。

### 1.2 软件环境

| 项目 | 版本/内容 |
|---|---|
| Ubuntu | **20.04.5 LTS** (aarch64) |
| ROS | **ROS 2 Foxy** (在 Docker 容器内运行) |
| 开发语言 | Python、C++ |
| 底盘开发 | C (AT32 MCU) |
| APP 开发 | 鸿蒙 ArkTS |

### 1.3 外观尺寸

| 参数 | 值 |
|---|---|
| 尺寸 | L400 × H320 × W320 mm |
| 离地间隙 | 20 mm |
| 自重 | 5 KG |
| 负载 | 7 KG |
| 电池 | DC 12V，12000 mAh |

## 2. 硬件规格与设备构成

iCar 智能小车整体构成分为：

1. **Jetson Orin Nano 算力开发板** — 主控计算平台
2. **思岚 A1 激光雷达 (RPLIDAR A1)** — 自主定位导航
3. **奥比中光 Astra Pro Plus 深度相机** — 视觉感知
4. **直流减速电机** (4个) — 动力装置
5. **AT32 开发板** — 底盘驱动控制
6. **环境感知模块** (8个 IOT 传感器) — GPS、NBIOT、WiFi、PM2.5、烟雾、温湿度、光照、大气压
7. **供电电源**

### 2.1 Jetson Orin Nano

| 规格 | 参数 |
|---|---|
| GPU | NVIDIA Ampere 架构，512 CUDA 核心 + 16 Tensor 核心 |
| CPU | 6核 Arm Cortex-A78AE @ 1.5GHz |
| AI 算力 | 20 TOPS |
| 内存 | 4GB / 8GB 64-bit LPDDR5，带宽 32GB/s |
| 存储 | 256GB NVMe SSD (M.2) |
| 功耗 | 7W ~ 10W 可配置 |
| USB | 4× USB 3.2 Gen2 Type-A，1× USB Type-C |
| 网络 | 千兆以太网，板载 802.11ac Wi-Fi + Bluetooth |
| 显示 | 1× DisplayPort 1.2 (支持 4K30) |

### 2.2 思岚 A1 激光雷达 (RPLIDAR A1)

| 规格 | 参数 |
|---|---|
| 测量半径 | 0.15 ~ 12.00 m |
| 测距精度 | ≤1% (≤3m)，≤2% (3~5m)，≤2.5% (5~12m) |
| 扫描频率 | 5.5 Hz (实测 10 Hz) |
| 扫描角度 | 360° |
| 测距频率 | 8000 Hz |
| 激光波长 | 785 nm |
| 通信接口 | UART (通过 USB 转串口) |
| 工作温度 | 0°C ~ 40°C |

### 2.3 奥比中光 Astra Pro Plus 深度相机

| 规格 | 参数 |
|---|---|
| 技术 | 单目结构光 3D 深度相机 |
| 工作范围 | 0.6 ~ 8 米 (室内) |
| 深度分辨率 | 640 × 480 @ 30Hz |
| 深度精度 | < ±3mm (1m 内) |
| IR 分辨率 | 640 × 480 @ 30Hz |
| 接口 | USB 2.0 |

### 2.4 直流减速电机

高精度大扭矩 520 编码直流电机，自带 AB 相增量式霍尔编码器，全金属齿轮。

### 2.5 AT32 控制板

国产雅特力 AT32F403A，基于 ARM Cortex-M4 内核，主频 240MHz，与 STM32F103 系列 PIN TO PIN 兼容。

### 2.6 设备映射与检查

小车启动后自动创建以下设备链接：

| 设备链接 | 目标 | 用途 |
|---|---|---|
| `/dev/myserial` | `/dev/ttyUSB1` | 底盘串口 (AT32 控制板) |
| `/dev/rplidar` | `/dev/ttyUSB0` | 激光雷达 (RPLIDAR A1) |
| `/dev/astradepth` | USB 设备 (ID 2bc5) | 深度相机 |
| `/dev/astrauvc` | USB 设备 (ID 2bc5) | 相机 UVC |
| `/dev/video0` | USB 摄像头 | 视频设备 |

```bash
# 检查设备
ls -la /dev/ttyUSB*
ls -la /dev/myserial /dev/rplidar /dev/astradepth /dev/astrauvc
lsusb                              # 应看到两行 2bc5 (Astra 相机)
```

## 3. 设备操作

### 3.1 设备充电

使用配套充电器连接小车充电口。充电时指示灯红色，充满后变为绿色。

> ⚠️ 小车电量不足时会运行缓慢，使用前记得充电。

### 3.2 设备上电

1. 确保底盘控制线正确连接
2. 按下面板上的电源按钮
3. 等待 Jetson 系统启动完成

### 3.3 设备联网

小车支持以下联网方式：

- **WiFi**: 板载 802.11ac 无线网卡
- **有线**: 千兆以太网 RJ45 接口
- **远程连接**: 使用笔记本电脑通过网络 SSH 连接小车

## 4. Docker 容器管理

小车使用 Docker 容器运行 ROS 2 环境。当前使用的镜像为 `icar/ros-foxy:1.0.2`。

### 4.1 查看容器

```bash
# 查看所有容器
sudo docker ps -a

# 查看正在运行的容器
sudo docker ps
```

### 4.2 启动/停止/登入容器

建议在 `~/.bashrc` 中配置快捷指令：

```bash
# 在 ~/.bashrc 中添加 (以容器 ID 前缀 cb62 为例)
alias s='docker start cb62'          # 启动容器
alias t='docker stop cb62'           # 停止容器
alias d='docker exec -it cb62 /bin/bash'  # 登入容器
```

配置后执行 `source ~/.bashrc` 生效。

### 4.3 进入容器

```bash
# 直接登入
sudo docker exec -it cb62aa7f1d8b /bin/bash

# 或使用快捷指令 (如已配置)
d
```

### 4.4 容器内加载环境

进入容器后，**每次都要先加载 ROS 工作区**：

```bash
source /root/icar_ros2_ws/icar_ws/install/setup.bash
```

建议加到容器内的 `~/.bashrc` 中以避免重复输入。

## 5. 容器内快捷指令

容器 (`icar/ros-foxy:1.0.2`) 内预定义了以下快捷指令 (在 `/root/.bashrc` 中)：

| 快捷指令 | 完整指令 | 用途 |
|---|---|---|
| `n1` | `ros2 run icar_bringup Mcnamu_driver_X3` | 启动小车底盘驱动 |
| `n2` | `ros2 launch sllidar_ros2 sllidar_launch.py` | 启动激光雷达 |
| `m1` | `ros2 launch astra_camera astra.launch.xml` | 启动深度相机 |

### 激光雷达应用

| 快捷指令 | 完整指令 | 用途 |
|---|---|---|
| `n3` | `ros2 run icar_laser laser_Avoidance_a1_X3` | 启动雷达避障 ⚠️ |
| `n4` | `ros2 run icar_laser laser_Tracker_a1_X3` | 启动雷达跟随 ⚠️ |
| `n5` | `ros2 run icar_laser laser_Warning_a1_X3` | 启动雷达警卫 |

### 视觉应用

| 快捷指令 | 完整指令 | 用途 |
|---|---|---|
| `m2` | `ros2 run icar_astra colorHSV` | HSV 颜色分析 |
| `m3` | `ros2 run icar_astra colorTracker` | 颜色/目标追踪 ⚠️ |

> ⚠️ 标记表示小车会自主移动，请注意安全。

## 6. 启动各项功能

### 6.1 底盘驱动 (必须最先启动)

```bash
source /root/icar_ros2_ws/icar_ws/install/setup.bash
ros2 run icar_bringup Mcnamu_driver_X3
```

**启动后发布的话题：**

| 话题 | 类型 | 说明 |
|---|---|---|
| `/cmd_vel` | `Twist` | 速度控制指令 (订阅) |
| `/imu/data_raw` | `Imu` | IMU 惯性测量数据 |
| `/imu/mag` | `MagneticField` | 磁场数据 |
| `/voltage` | `Float32` | 电池电压 |
| `/Buzzer` | `Bool` | 蜂鸣器控制 |
| `/RGBLight` | `Int32` | RGB 灯效控制 |
| `/vel_raw` | `Twist` | 原始速度反馈 |
| `/joint_states` | `JointState` | 关节状态 |
| `/edition` | `Float32` | 版本信息 |

> 启动后小车不会自动移动，需要发 `/cmd_vel` 指令才会动。

### 6.2 激光雷达

```bash
source /root/icar_ros2_ws/icar_ws/install/setup.bash
ros2 launch sllidar_ros2 sllidar_launch.py
```

**参数：** 扫描距离 12m，频率 10Hz，采样率 8KHz  
**话题：** `/scan` (`LaserScan`)

### 6.3 深度相机 (Astra Pro Plus)

```bash
source /root/icar_ros2_ws/icar_ws/install/setup.bash
ros2 launch astra_camera astra.launch.xml
```

**发布的话题：**

| 话题 | 类型 | 说明 |
|---|---|---|
| `/camera/depth/image_raw` | `Image` | 深度图像 (640×480，16UC1) |
| `/camera/depth/points` | `PointCloud2` | 点云数据 |
| `/camera/depth/camera_info` | `CameraInfo` | 深度相机参数 |
| `/camera/ir/image_raw` | `Image` | 红外图像 (640×480) |
| `/camera/ir/camera_info` | `CameraInfo` | IR 相机参数 |

> 当前配置下颜色流默认未启用。如需启用：`ros2 launch astra_camera astra.launch.xml enable_color:=true`

## 7. 常用 ROS 2 命令

### 查看状态

```bash
# 列出所有话题
ros2 topic list

# 列出所有话题 (含类型)
ros2 topic list -t

# 查看话题信息
ros2 topic info /scan

# 查看发布频率
ros2 topic hz /scan

# 查看节点信息
ros2 node info /camera/camera

# 查看所有节点
ros2 node list
```

### 手动控制小车移动

> 💡 发送一次指令小车会持续运动，直到收到新的指令。记得发停止指令。

```bash
# 前进 (速度 0.1 m/s)
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.1, y: 0.0, z: 0.0}, angular: {z: 0.0}}"

# 后退
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: -0.1, y: 0.0, z: 0.0}, angular: {z: 0.0}}"

# 左平移 (麦克纳姆轮)
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.0, y: 0.1, z: 0.0}, angular: {z: 0.0}}"

# 右平移
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.0, y: -0.1, z: 0.0}, angular: {z: 0.0}}"

# 左转
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {z: 0.3}}"

# 右转
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {z: -0.3}}"

# 停止
ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {z: 0.0}}"
```

### 查看数据

```bash
# 电池电压
ros2 topic echo /voltage

# IMU 数据
ros2 topic echo /imu/data_raw

# 激光雷达数据
ros2 topic echo /scan

# 深度图像
ros2 topic echo /camera/depth/image_raw

# 点云
ros2 topic echo /camera/depth/points
```

## 8. 功能详解

### 8.1 激光雷达自动避障 ⚠️

```bash
# 前提：n1 (底盘) + n2 (雷达) 已启动
ros2 run icar_laser laser_Avoidance_a1_X3
```

小车会自主运动，并能自主规避前进中的障碍物，避免发生碰撞。

### 8.2 激光雷达自动跟随 ⚠️

```bash
# 前提：n1 (底盘) + n2 (雷达) 已启动
# 先按 Ctrl+C 停止避障，再执行：
ros2 run icar_laser laser_Tracker_a1_X3
```

人在小车前面走动，小车会跟随人前进或后退以保持固定距离。

### 8.3 激光雷达警卫

```bash
# 前提：n1 (底盘) + n2 (雷达) 已启动
# 先按 Ctrl+C 停止跟随，再执行：
ros2 run icar_laser laser_Warning_a1_X3
```

小车会保持盯着人的方向。如果人太过靠近小车，小车会发出蜂鸣器告警声。

### 8.4 目标追踪 (视觉颜色追踪) ⚠️

需要 3 个终端 (通过 `id` 登入容器)：

```bash
# 终端 1：底盘驱动
n1

# 终端 2：深度相机
m1

# 终端 3：HSV 颜色分析 (弹出分析窗口)
m2

# 终端 4：目标追踪
m3
```

在 m2 程序界面按下空格键，小车开始追踪测试人员手中的目标物。目标物移动时，小车会前后移动或左右转向，追踪目标物并保持一定距离。

### 8.5 自动循线 (需刷写固件)

自动循线功能需要烧写专门的底盘固件 `follow-line.bin`：

1. 使用 ArteryISPProgrammer 将 `follow-line.bin` 烧写入 AT32 底盘控制板
   - 将 BOOT1 按钮拨到 VDD
   - 连接 USB 到电脑，运行烧录软件
   - 烧写完成后将 BOOT1 拨回 GND
   - 将 USB 线插回小车顶部中间位置

2. 通过 VNC 远程到小车，启动 app 程序

3. 将小车放置在循线地图上，勾选 "循迹指令"

4. 小车将沿着黑线自主行驶

5. 结束循线后，将底盘固件刷回非循线版本 `iCar.bin`

## 10. 注意事项

1. **充电**：小车电量不足时会运行缓慢，使用前记得充电。充电器指示灯变绿表示充满电。

2. **连接检查**：小车使用前，确保底盘控制线的连接正确。

3. **安全测试**：
   - 测试避障、跟随、追踪等功能时，建议在开阔空间进行
   - 速度设定不宜过快
   - 注意防止小车撞到周围的障碍物
   - 为了安全起见，可以用支撑物将小车支撑起来，四轮悬空进行测试

4. **循线固件**：循线完成后，记得将底盘固件刷回非循线版本 `iCar.bin`。

5. **相机连接**：若 `lsusb` 只看到一行 `2bc5`，需要重新拔插深度相机的 USB 连线。

## 11. 故障排除

### 11.1 串口设备找不到

```bash
# 检查设备是否存在
ls -la /dev/ttyUSB*
ls -la /dev/myserial /dev/rplidar

# 检查权限
sudo chmod 666 /dev/ttyUSB*
```

### 11.2 容器无法启动

```bash
# 查看错误日志
sudo docker logs cb62aa7f1d8b

# 检查设备映射
sudo docker inspect cb62aa7f1d8b | grep -A5 Devices
```

### 11.3 激光雷达无数据

```bash
# 检查设备
ls -la /dev/rplidar

# 检查话题
ros2 topic echo /scan
```

### 11.4 摄像头无图像

```bash
# 检查设备 (应看到两行 2bc5)
lsusb | grep 2bc5

# 若只有一行，重新拔插深度相机 USB 线
ls -la /dev/astradepth /dev/astrauvc

# 查看相机信息
ros2 run astra_camera list_devices_node
```

### 11.5 ROS 命令找不到

```bash
# 确保在容器内且已加载环境
source /root/icar_ros2_ws/icar_ws/install/setup.bash
```
