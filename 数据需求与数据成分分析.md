## 二、传感器数据

### 2.1 原始数据包格式（串口，115200 baud）

| 字节偏移 | 大小 | 字段 | 说明 |
|----------|------|------|------|
| 0 | 1 | `header` | 固定 `0xA5` |
| 1 | 1 | `node` | 节点号 |
| 2 | 1 | `type` | 传感器类型码 |
| 3–4 | 2 | `value` | uint16, little-endian |
| 5 | 1 | `checksum` | `(0xA5 + data[1..4]) & 0xFF` |

### 2.2 传感器类型码与值域

| 码 | 名称 | 解析公式 | 单位 | 样例值 |
|----|------|----------|------|--------|
| `0x01` | `temperature` | raw / 10.0 | °C | 263 → 26.3°C |
| `0x02` | `humidity` | raw / 10.0 | % | 550 → 55.0% |
| `0x03` | `smoke` | raw | — | 0 |
| `0x04` | `pm25` | raw | μg/m³ | 35 |
| `0x05` | `pressure` | raw | hPa | 1013 |
| `0x06` | `light` | raw | lux | 500 |
| `0x07` | `gps_lat` | raw (float) | 度 | 39.9 |
| `0x08` | `gps_lon` | raw (float) | 度 | 116.4 |
| `0x09` | `co2` | raw | ppm | 400 |

### 2.3 解析后状态结构

```json
{
  "temperature": 25.0,
  "humidity": 50.0,
  "smoke": 0,
  "pm25": 0,
  "pressure": 1013,
  "light": 500,
  "co2": 400,
  "gps": {"lat": 0.0, "lon": 0.0}
}
```

### 2.4 异常数据需求

| 异常 | 表现 | 当前处理 | 数据需求 |
|------|------|----------|----------|
| 传感器离线 | 数据永不更新 | 返回旧值 | 需增加 `_stale_seconds` 字段标记过期 |
| 校验和错误 | `parse()` → None | 静默丢弃 | 需增加 `parse_errors` 计数 |
| 传感器值超量程 | 如 9999 | 无校验 | 需增加合法值域校验 |

## 三、烟火检测核心数据链

### 3.1 状态机数据流

```
YOLO逐帧检测 (Detection[])
  → 命中窗口累计 (置信度 ≥ 0.70, 2秒窗口内 ≥ 5次命中)
  → 创建事件 (_Event, event_id 格式: fire_YYYYMMDD_HHMMSS_µs_序号)
  → IDLE → AI_REVIEWING
      ↓
    抓拍原始帧 JPEG → AIReviewRequest (event_id + base64)
      ↓
    云端 GPT 视觉复核 (30s 超时, 重试 2 次)
      ↓
    AIResultKind: confirmed_fire / suspected_smoke / no_fire
      ↓
    ┌─ confirmed_fire  → ALARMED_FIRE  → AlarmEvent → 上报
    ├─ suspected_smoke → ALARMED_SMOKE → AlarmEvent → 上报
    ├─ no_fire         → AI_REJECTED   → 30s 后重新复审
    └─ 全部失败        → AI_FAILED     → ai_unavailable 告警
```

### 3.2 证据文件对结构 (JPEG + JSON)

**文件命名**: `{event_id}_{review_id:03d}_{capture_type}_{HHMMSSffffff}.jpg`

**JSON 元数据**:
```json
{
  "event_id": "fire_20260712_094811_058310_001",
  "review_id": 1,
  "capture_type": "initial|review|periodic",
  "captured_at": "2026-07-12T09:48:11.058310+08:00",
  "local_detection": {
    "classes": ["fire", "smoke"],
    "max_confidence": 0.6577,
    "trigger_hits": 5,
    "window_seconds": 2.0
  },
  "ai_review": {
    "status": "pending|completed|failed|not_requested",
    "attempts": 0,
    "result": "confirmed_fire|null",
    "confidence": 0.98,
    "reason": "图像中可见明亮火焰",
    "error": ""
  }
}
```

**捕获类型**:
| 类型 | 含义 |
|------|------|
| `initial` | 触发时的第一帧（带标注框） |
| `review` | AI 复核后重新触发 |
| `periodic` | 已报警后的 60s 定期证据 |

**存储**: `fire_smoke_detection/runtime/evidence/` | 轮转上限: 10 对

### 3.3 AI 复核请求/响应格式

**请求** (OpenAI Chat Completions API):
```json
{
  "model": "gpt-5.4-mini",
  "temperature": 0,
  "messages": [{
    "role": "user",
    "content": [
      {"type": "text", "text": "你是火灾/烟雾巡检机器人..."},
      {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,<b64>"}}
    ]
  }]
}
```

**响应 JSON**:
```json
{"result": "confirmed_fire", "confidence": 0.98, "reason": "图像中可见明亮火焰区域"}
```

### 3.4 告警上报数据

**告警 JSON** (POST 到云端):
```json
{
  "event_id": "fire_20260712_143052_123456_001",
  "alarm_type": "confirmed_fire|suspected_smoke|ai_unavailable",
  "occurred_at": "2026-07-12T14:30:52.123456+08:00",
  "reason": "AI 复核原因",
  "confidence": 0.91,
  "evidence_url": "http://8.140.28.233/evidence/fire_xxx.jpg",
  "detection_classes": "fire,smoke",
  "max_confidence": 0.88,
  "local_detection_gone": false,
  "car_id": "car-01"
}
```

**证据图片上传**: multipart form-data → `POST /api/v1/evidence` → 返回 `{"code": 0, "data": {"url": "http://.../证据.jpg"}}`

## 四、云端数据库

### 4.1 `fire_alarm` 表 (MySQL, `dev_fireguard` 库, utf8mb4)

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| `id` | `BIGINT` | PK, AUTO_INCREMENT | 自增主键 |
| `event_id` | `VARCHAR(64)` | NOT NULL, UNIQUE (联合) | 车端事件 ID |
| `alarm_type` | `VARCHAR(32)` | NOT NULL, UNIQUE (联合) | confirmed_fire / suspected_smoke / ai_unavailable |
| `occurred_at` | `DATETIME(6)` | NOT NULL, INDEXED | 事件发生时刻（微秒） |
| `reason` | `VARCHAR(300)` | NULL | AI 复核原因 |
| `confidence` | `DECIMAL(4,3)` | NULL | AI 置信度 (0.000–1.000) |
| `evidence_url` | `VARCHAR(500)` | NULL | 证据图片 URL |
| `detection_classes` | `VARCHAR(200)` | NULL | 逗号分隔 YOLO 命中类别 |
| `max_confidence` | `DECIMAL(4,3)` | NULL | YOLO 最大置信度 |
| `local_detection_gone` | `TINYINT(1)` | NULL | 本地检测是否消失 |
| `car_id` | `VARCHAR(64)` | NULL, INDEXED | 车辆标识 |
| `received_at` | `DATETIME(6)` | NOT NULL | 云端入库时刻 |
| `raw_payload` | `JSON` | NULL | 原始上报 JSON 完整存档 |

**索引**:
- `PRIMARY KEY (id)`
- `UNIQUE KEY uk_event_type (event_id, alarm_type)` — 幂等去重
- `KEY idx_occurred (occurred_at)`
- `KEY idx_type (alarm_type)`
- `KEY idx_car (car_id)`

**书写方式**: `INSERT ... ON DUPLICATE KEY UPDATE id = LAST_INSERT_ID(id)` — 支持重复上报幂等

### 数据流方向总结

| 流向 | 数据 | 协议 |
|------|------|------|
| **传感器 → 车 API** | 传感器状态字典 | 内存 (线程共享) |
| **YOLO → EventManager** | `Detection[]` | 内存 (函数调用) |
| **EventManager → AIReviewer** | `AIReviewRequest` (JPEG bytes) | 内存 (Queue) |
| **AIReviewer → 外部 AI** | OpenAI Vision API (base64 JPEG) | HTTPS |
| **EventManager → EvidenceStore** | 标注帧 + 元数据 | 文件系统 (JPEG+JSON 成对) |
| **EventManager → AlarmService** | `AlarmEvent` | 内存 |
| **CloudAlarmService → 云端** | 证据 JPEG (multipart) + 告警 JSON | HTTP POST |
| **CloudAlarmService → outbox** | 告警 JSON (失败时) | 文件追加 (JSONL) |
| **车 API → Coordinator** | 状态 JSON | HTTP GET (轮询 5s) |
| **Coordinator → 车 API** | move/navigate 命令 | HTTP POST |
| **语音服务 → 车** | 录音/播放命令 | HTTP (车端音频接口) |
| **语音服务 → 外部** | ASR音频/LLM对话/TTS请求 | HTTPS |
| **云端 → MySQL** | `fire_alarm` 行 | PyMySQL |
| **云端 → Web 前端** | `Alarm[]` JSON | HTTP GET (REST) |

## 十、数据异常需求 (补充)

基于异常流程分析，当前系统缺失以下数据需求：

| # | 数据需求 | 用途 | 当前状态 |
|---|----------|------|----------|
| 1 | **传感器陈旧标志** `_stale_seconds` | 判断数据是否过期 | ❌ 缺失 |
| 2 | **传感器解析错误计数** | 诊断串口链路异常 | ❌ 缺失 |
| 3 | **AI Reviewer 心跳时间戳** | 检测 Worker 线程存活 | ❌ 缺失 |
| 4 | **YOLO 进程存活标志** | 检测核心检测进程崩溃 | ❌ 缺失 |
| 5 | **告警送达确认回执** | 防止火警漏报 | ❌ 缺失 |
| 6 | **底盘运动状态反馈** | 确认指令已执行 | ❌ 缺失 (命令发送即返回成功) |
