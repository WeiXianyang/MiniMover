# MiniMover 烟火识别模块

该模块使用迁移后的旧版 YOLOv5 和训练模型 `model/best.pt` 识别 `fire` 与 `smoke`。它作为独立模块运行，不修改 MiniMover 的车牌识别、红绿灯识别、Web 控制或小车控制代码。

## Windows 快速开始

```powershell
cd E:\MiniMover\fire_smoke_detection
python -m pip install -r requirements.txt
run.bat --source 0 --view-img
run.bat --source samples\result_demo.jpg
```

## Linux / Jetson 快速开始

Jetson 应先安装与 JetPack/CUDA 版本匹配的 NVIDIA PyTorch 和 torchvision，不要用通用 CUDA wheel 覆盖 JetPack 配套版本。随后安装其余依赖：

```bash
cd /path/to/MiniMover/fire_smoke_detection
python3 -m pip install -r requirements.txt
bash run.sh --source 0 --view-img
bash run.sh --source samples/result_demo.jpg
```

也可以通过 `PYTHON_BIN` 指定解释器：

```bash
PYTHON_BIN=/usr/bin/python3 bash run.sh --source 0 --device 0 --view-img
```

## 通用命令

```bash
python detector.py --source 0 --view-img
python detector.py --source /path/to/image.jpg
python detector.py --source /path/to/video.mp4
python detector.py --source 0 --device cpu --conf-thres 0.4
python detector.py --source 0 --device 0 --iou-thres 0.5
```

相对输入路径按调用命令时的当前目录解析。模型和输出目录始终按模块位置解析，因此不要求从模块目录启动。结果默认写入 `output/`。旧版检测器每次运行会重新创建该目录，不要将长期保存的证明材料放入其中。

## 类别

- `fire`
- `smoke`

## 目录说明

- `model/`：训练后的烟火识别模型。
- `yolov5_runtime/`：与模型匹配的旧版 YOLOv5 推理源码。
- `training/VOC2020/`：完整训练图片、XML 标注、YOLO 标签和数据划分。
- `training/runs/`：训练、验证和检测记录，包括曲线、批次图和权重。
- `training/scripts/`：训练辅助脚本。
- `training/source/`：训练源码快照和原始绝对路径数据配置。
- `training/fire_smoke.yaml`：迁移后的相对路径数据配置。
- `training/original_readmes/`：原项目说明和许可证。
- `evidence/`：原 Git 历史、未提交修改、文件清单、哈希和迁移报告。
- `samples/`：冒烟测试素材。
- `output/`：可删除的运行输出，不纳入 Git。

## 训练复现

迁移后的数据配置为 `training/fire_smoke.yaml`。原训练源码和依赖版本较旧，复现实验时应优先采用原项目记录对应的 Python、PyTorch 与 CUDA 环境。原始绝对路径配置保存在 `training/source/fire_smoke.original.yaml`，仅用于证明迁移前状态。

## Jetson 注意事项

本次迁移保持旧版 PyTorch/YOLOv5 推理链路，不包含 ONNX 或 TensorRT 转换。Jetson GPU 推理需要根据实际 JetPack 版本安装兼容的 PyTorch、torchvision 和 OpenCV。

## 来源与许可证

原项目文档和许可证位于 `training/original_readmes/`。完整 Git 历史归档、迁移前工作区补丁及来源记录位于 `evidence/`。
