# 项目启动说明

本文档说明如何在华为模拟器上编译、安装并启动本 OpenHarmony 小车遥控应用。

## 一、关键信息

| 项目 | 值 |
| --- | --- |
| 工程路径 | `C:\Users\32599\Downloads\oh-ai-car-ros-app\oh-ai-car-ros-app` |
| hdc 工具 | `D:/open/23/toolchains/hdc.exe`（注意路径中带 API 版本号 `23`） |
| 模拟器地址 | `127.0.0.1:5555` |
| HAP 产物 | `entry/build/default/outputs/default/entry-default-signed.hap` |
| 包名 bundleName | `com.hoperun.cmartcar` |
| 启动 Ability | `EntryAbility` |
| SDK 目录 | `D:/open`（standalone OpenHarmony SDK，API 23） |
| Node 目录 | `D:/DevEco Studio/tools/node` |

## 二、前置条件

1. 已在 DevEco Studio 中启动华为模拟器（Huawei Emulator）。
2. 已完成签名配置（DevEco 自动生成签名）。
3. 首次编译前需保证 `hvigor/hvigor-config.json5` 与 `build-profile.json5` 配置正确（见下文说明）。

## 三、日常启动（已编译过，只需重装重启）

已经编译出 `entry-default-signed.hap` 后，重启应用只需三步：

```bash
# 1. 连接模拟器
"D:/open/23/toolchains/hdc.exe" tconn 127.0.0.1:5555

# 2. 安装 HAP（先 cd 到产物目录，避免 hdc 路径拼接 bug）
cd "entry/build/default/outputs/default"
"D:/open/23/toolchains/hdc.exe" install entry-default-signed.hap

# 3. 启动应用
"D:/open/23/toolchains/hdc.exe" shell aa start -a EntryAbility -b com.hoperun.cmartcar
```

看到 `start ability successfully.` 即表示启动成功。

> 注意：`hdc install` 传绝对路径会出现路径重复拼接的问题，务必先 `cd` 到 HAP 所在目录，再用相对文件名安装。

## 四、完整编译并启动（从源码开始）

在 DevEco Studio 中：

1. 打开工程 `oh-ai-car-ros-app`。
2. 执行 `Build > Build Hap(s)/APP(s) > Build Hap(s)` 生成签名 HAP。
3. 编译成功后，按第三节步骤安装并启动。

或使用命令行编译后再执行第三节步骤。

## 五、常见问题排查

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `hdc.exe: No such file or directory` | hdc 路径错误 | 使用 `D:/open/23/toolchains/hdc.exe`，路径带版本号 `23` |
| `hdc list targets` 为空 | 模拟器未连接 | 执行 `hdc tconn 127.0.0.1:5555` |
| install 报路径找不到 | 绝对路径被重复拼接 | 先 `cd` 到 HAP 目录再用相对文件名安装 |
| `Target is connected, repeat operation` | 已连接 | 正常，可直接安装 |
| 编译报 SDK 组件找不到（00308018） | SDK 布局不匹配 | 工程已降级到 API 23 匹配 `D:/open` |
| 编译报签名失败（00303107） | 未配置签名 | 用 DevEco 自动生成签名 |

## 六、验证启动是否成功

```bash
# 查看应用是否已安装
"D:/open/23/toolchains/hdc.exe" shell bm dump -a | findstr cmartcar

# 查看运行日志
"D:/open/23/toolchains/hdc.exe" shell hilog | findstr cmartcar
```
