## 二、业务进程详细分析

### 2.1 系统启动与自检

| 维度 | 内容 |
|------|------|
| **Actors** | 小车（Jetson Orin Nano）、底盘MCU（AT32）、传感器模组、摄像头、操作员 |
| **Trigger** | 小车开机通电 / 执行 `start_services.sh` 或 `python api_server.py` |
| **Main Flow** | ① 传感器驱动初始化（9种传感器串口 `/dev/ttyUSB2`）→ ② 底盘驱动初始化（Rosmaster 串口 `/dev/myserial`，电池电压自检）→ ③ 摄像头初始化（深度相机 `/dev/camera_*`，MJPEG 编码就绪）→ ④ YOLOv5 烟火检测模型加载（`model/best.pt`）→ ⑤ Flask API 服务启动（端口 5000）→ ⑥ 多车协同调度中心就绪（端口 8888）→ ⑦ ROS2 Docker 容器启动（SLAM/Nav2/激光雷达节点） |
| **End State** | 全部子系统就绪：API 端口可访问、传感器数据可读取、视频流可订阅、检测模型运行中 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 串口打开失败（`/dev/ttyUSB2` 不存在） | `connect()` 返回 False，API 返回默认值，程序不退出 | 🔴 Critical |
| 底盘 MCU 无响应 | `set_car_motion()` 无 try/except 保护 | 🔴 Critical |
| 摄像头初始化失败（`/dev/camera_*` 不存在） | `isOpened()` → False，视频流返回空字节 | 🟠 Major |
| 模型文件缺失（`model/best.pt` 不存在） | YOLOv5 进程退出，无守护进程 | 🔴 Critical |
| CUDA/GPU 异常 | 程序崩溃无告警 | 🔴 Critical |

### 2.2 巡检任务执行

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员（发起任务）、小车（自主导航+检测）、AI复核服务、云端平台 |
| **Trigger** | 操作员通过 Web/App 面板下发巡检任务 或 预设定时任务触发 |
| **Main Flow** | ① 操作员设定巡检路线（预设航点/区域）→ ② 小车启动 SLAM 自主导航（Nav2 路径规划 + 激光雷达避障）→ ③ 小车沿路线行驶，深度相机持续采集画面 → ④ YOLOv5 逐帧推理检测 fire/smoke（置信度 ≥ 0.70）→ ⑤ 9 种 IoT 传感器持续采集环境数据 → ⑥ 视频流实时回传操作员控制台 → ⑦ 巡检完成/电量不足时自动返航 → ⑧ 巡检日志写入本地 `runtime/fire_monitor.log` |
| **End State** | 巡检路线完成，操作员获得完整巡检报告 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 摄像头运行中断连（USB 松动） | `get_frame()` 返回空字节 → YOLO 输入为空 → 检测停摆 | 🟠 Major |
| ROS2 容器未运行 | 视频代理 8080 端口无响应 → 静默等待 | 🟠 Major |
| SLAM 导航定位丢失 | 小车原地旋转尝试重定位 | 🟠 Major |
| 串口写入失败 | MCU 无响应，但 API 仍返回 success → 操作员无感知 | 🔴 Critical |
| 导航命令失败 | stdout/stderr 全部丢弃到 DEVNULL | 🔴 Critical |

### 2.3 烟火检测与告警生命周期（核心）

这是系统最核心的业务流程，涉及从 YOLO 初检到 AI 复核到告警上报的完整状态机。

**状态机：**
```
IDLE → AI_REVIEWING → ALARMED_FIRE / ALARMED_SMOKE / AI_REJECTED / AI_FAILED → IDLE
```

| 步骤 | 状态 | 动作 |
|------|------|------|
| 1 | **IDLE** | YOLO 逐帧检测，置信度 ≥ 0.70 的 fire/smoke 判定为"命中"，时间戳压入滑动窗口 |
| 2 | **IDLE** | 窗口内命中数 ≥ 5（2 秒窗口）→ 创建事件 `fire_YYYYMMDD_HHMMSS_µs_序号`，状态切为 AI_REVIEWING |
| 3 | **AI_REVIEWING** | 抓拍未画框原始 JPEG，保存到 `runtime/evidence/` + 元数据 JSON |
| 4 | **AI_REVIEWING** | 图片 Base64 编码后提交云端 AI 复核（GPT 视觉模型，Bearer Token 鉴权） |
| 5 | **判决** | AI 返回结果 → 进入对应状态 |

**分支流程：**

| AI 结果 | 目标状态 | 动作 |
|---------|----------|------|
| `confirmed_fire` | **ALARMED_FIRE** | 触发火灾报警 → AlarmService 上报 |
| `suspected_smoke` | **ALARMED_SMOKE** | 触发烟雾报警 → AlarmService 上报 |
| `no_fire` | **AI_REJECTED** | 不报警；30 秒后可再次复审 |
| 全部失败（重试 2 次） | **AI_FAILED** | 触发 ai_unavailable 告警 → 提示人工介入 |

**告警状态下的持续动作：**
- 每 60 秒自动保存一张带标注框的证据图（最多 10 张）
- 事件清除：YOLO 目标消失超过 10 秒且无 in_flight 任务 → 回退 IDLE

**关键配置阈值：**

| 参数 | 默认值 |
|------|--------|
| YOLO 置信度阈值 | 0.70 |
| 触发窗口 | 2 秒 / 5 次命中 |
| AI 请求超时 | 30 秒 |
| AI 重试次数 | 2 次（共 3 次） |
| AI_REJECTED 复审间隔 | 30 秒 |
| 周期性证据间隔 | 60 秒 |
| 最大证据图片 | 10 张 |
| 事件自动清除 | 10 秒 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| **告警上报失败但状态已标记 ALARMED** | alarm 设为 None，状态推进 → 车端认为已告警，实际未送达 | 🔴 Critical |
| AI API Key 未配置 | 返回 success=False → AI_FAILED | 🔴 Critical |
| 网络超时（30s） | 立即重试无退避 → AI_FAILED | 🟠 Major |
| AI 返回无效 JSON | `parse_model_content()` → 计入重试次数 | ✅ 已处理 |
| AI 复核队列满 | `submit()` 返回 False → RuntimeError → 事件丢失 | 🟠 Major |
| JPEG 编码失败 | RuntimeError → 事件中断 | 🟠 Major |
| Worker 线程死亡 | Daemon 无监控/无自动重启 | 🟠 Major |
| YOLO 检测进程崩溃 | subprocess 返回非零 → 无人感知 | 🔴 Critical |

### 2.4 告警确认与人工处置

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员（值班人员/管理员）、云端平台、小车 |
| **Trigger** | 操作员收到告警通知（Web 自动刷新 / App 推送） |
| **Main Flow** | ① 操作员收到告警 → ② 打开告警详情页，查看证据图片、AI 结论、置信度、检测数据 → ③ 执行处置动作：`acknowledge`（确认收到）/ `resolve`（已处理）/ `false_alarm`（确认误报）/ `escalate`（升级）→ ④ 系统记录操作人、时间、动作到审计表 → ⑤ 若需远程处置小车：下发 `stop_patrol` / `start_recording` / `open_live_view` / `return_home` → ⑥ 小车接收命令并执行回执 |
| **End State** | 告警被人工确认/处置完毕，审计记录完整可追溯 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 网关不可达 | 告警写入 `outbox.jsonl` 离线队列，指数退避重试 | ✅ 已处理 |
| App 断线恢复 | 重新拉取未关闭告警列表 | ✅ 已处理 |
| 重复上报 | 云端 `(event_id, alarm_type)` 幂等去重 | ✅ 已处理 |
| 云端 401 鉴权失败 | 写入 outbox → 无限重试（永远失败） | 🟠 Major |

### 2.5 多车协同编队调度

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员（指挥者）、多辆小车、协调器（端口 8888） |
| **Trigger** | 操作员通过调度中心面板发起编队控制 |
| **Main Flow** | ① 调度中心启动 → ② 后台轮询线程每 5 秒并行查询所有车辆状态 → ③ 状态缓存更新 → ④ 基于位置自动碰撞检测（< 0.5m CRITICAL，< 1.0m WARNING）→ ⑤ 操作员通过可视化面板查看 → ⑥ 下发队形命令（line / wedge / diamond）+ 间距 → ⑦ 调度中心计算目标位置 → 并行下发移动指令 |
| **End State** | 车辆编队到位，碰撞风险消除 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 车辆 IP 不可达 | `api()` 返回 `{'code': -1}`，不阻塞其他车辆 | ✅ 已处理 |
| 视频代理离线 | 返回占位图 "Car X Camera Offline" | ✅ 已处理 |
| **碰撞检测触发但无自动制动** | 仅设置 WARNING/CRITICAL 告警前端显示 | 🟠 Major |
| 语音进程异常退出 | `poll()` 检测，无自动重启 | 🟠 Major |
| 注册已存在车辆 | 返回 409 冲突 | ✅ 已处理 |

### 2.6 语音控制交互

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员（说话者）、语音助手服务、ASR 引擎（FunASR/Whisper）、LLM、TTS、小车底盘 |
| **Trigger** | 说出唤醒词 "你好小南" 或直接说出运动控制命令 |

**Main Flow（唤醒词模式）：**
```
① 持续监听 → ② 检测唤醒词 → ③ TTS 问候"你好，我是你好小南" 
→ ④ 进入 30s 活跃窗口 → ⑤ ASR 识别语音 → ⑥ command_parser 匹配命令
→ ⑦ 声纹验证（CAM++，阈值 0.38）→ ⑧ 转为 API 调用 POST /api/move
→ ⑨ 小车执行动作（含 0.5s 防抖）→ ⑩ 超时或说"停止" → 回退 IDLE
```

**语音命令词汇表：**

| 命令 | 触发词 |
|------|--------|
| `stop`（紧急） | 急停、停止、停下来、停下、别动、不要动 |
| `forward` | 向前、前进、往前走、直走 |
| `backward` | 向后转、后转、向后走、后退 |
| `left` | 向左转、左转、往左转、向左 |
| `right` | 向右转、右转、往右转、向右 |
| `spin` | 转圈、原地转、旋转一圈、转个圈 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 声纹验证失败（相似度 < 0.38） | 运动命令被阻止（安全导向） | ✅ 已处理 |
| 声纹未注册 | 拒绝所有运动命令 | ✅ 已处理 |
| TTS 请求失败 | 自动回退到 espeak-ng 离线 TTS | ✅ 已处理 |
| ASR 识别为空 | 忽略不处理 | ✅ 已处理 |
| FunASR 依赖缺失 | RuntimeError → 语音功能完全不工作 | 🔴 Critical |
| 音频设备不可用 | InputStream 抛异常 | 🟠 Major |
| 车控 HTTP 超时 | 捕获 + logging + TTS 反馈 | ✅ 已处理 |
| LLM 调用失败 | 捕获 + logging | ✅ 已处理 |

### 2.7 Web/App 监控面板交互

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员/管理员、Web 前端（React/TS + Tailwind）、云端 API（Flask + MySQL）、小车 API |
| **Trigger** | 操作员打开浏览器访问云端监控面板或直连小车 Web 页面 |

**Main Flow（云端告警面板）：**
```
① 访问 Overview → ② GET /api/v1/fire-alarms 加载列表 
→ ③ 展示：统计卡片 + 告警表格 + 趋势图 + 事件流 + 车辆状态
→ ④ 筛选（按类型/车辆/时间）→ ⑤ 点击详情 → 证据图 + AI结论 + 处置按钮
→ ⑥ 30s 自动刷新 → ⑦ API 失败自动降级 Mock 数据
```

**小车直连 Web：**
- `http://<car_ip>:6500` → Video 页面（MJPEG + 方向控制）
- `http://<car_ip>:5000/` → MiniMover 内嵌面板（方向键 + 传感器 + ping）

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| API 调用失败 | 错误横幅 + 降级到 Mock 数据 | ✅ 已处理 |
| 告警列表为空 | 空状态占位 "暂无告警数据" | ✅ 已处理 |
| 图片加载失败 | 占位图标 | 🟡 Minor |
| 数据库不可用 | `/healthz` → 500 | 🔴 Critical |

### 2.8 云端数据同步

| 维度 | 内容 |
|------|------|
| **Actors** | 车端 CloudAlarmService、云端 API（Flask + MySQL + Nginx） |
| **Trigger** | 车端产生确认告警事件 |
| **Main Flow** | ① `CompositeAlarmService` 串联 → ② 上传证据图片（multipart）→ ③ POST 告警 JSON → ④ 云端校验（Token + 输入 + 文件类型白名单 + 路径穿越防护）→ ⑤ MySQL 幂等写入 `INSERT ... ON DUPLICATE KEY` → ⑥ 图片存 `/data/fireguard/evidence/` → ⑦ 返回 `{"code":0}` |
| **End State** | 告警数据完整同步，证据图片可通过 HTTPS URL 访问 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 网络不可达 | outbox.jsonl 持久化 + 指数退避重试（2s → 300s） | ✅ 已处理 |
| 云端返回非 200 | 记录错误 → outbox 重试 | 🟠 Major |
| 证据上传失败 | 告警 JSON 中 `evidence_url` 为 null | 🟠 Major |
| outbox 写盘失败（磁盘满） | OSError 未被上层捕获 | 🟠 Major |
| 幂等去重 | 同一 `(event_id, alarm_type)` 不创建新记录 | ✅ 已处理 |

### 2.9 传感器数据采集与环境监控

| 维度 | 内容 |
|------|------|
| **Actors** | iCarSensorDriver、AT32 MCU、9 种 IoT 传感器、API 调用方 |
| **Trigger** | 系统启动时自动开始，周期性轮询 |
| **Main Flow** | ① 打开 `/dev/ttyUSB2`（115200 baud）→ ② 后台线程持续读取 6 字节帧 → ③ `parse()` 校验 → ④ 更新内存数据（温度/湿度/烟雾/PM2.5/气压/光照/CO₂/GPS）→ ⑤ `GET /api/sensors` 实时返回 |
| **End State** | 传感器数据持续更新 |

**异常流程：**

| 异常 | 处理 | 严重度 |
|------|------|--------|
| 串口打开失败 | API 返回默认值（温度 25°C、湿度 50%），程序不退出 | 🔴 Critical |
| 串口读取异常 | 裸 `except: time.sleep(0.5)` — 不记录日志、不自动重连 | 🟠 Major |
| 读取线程死亡 | 无监控、无自动重启 | 🟠 Major |
| 数据帧解析失败 | `parse()` → None，旧数据保持不变（可能过期） | 🟡 Minor |

### 2.10 语音播报与 TTS 服务

| 维度 | 内容 |
|------|------|
| **Actors** | 调用方、TTS 引擎（阿里云 CosyVoice 或 espeak-ng）、USB 音箱 |
| **Trigger** | `POST /api/audio/say` 或语音助手 LLM 回复 |
| **Main Flow** | ① 接收文本 → ② 阿里云 DashScope CosyVoice API（cosyvoice-v3-flash）→ ③ 获取 WAV bytes → ④ 增益 1.8x → ⑤ 重采样 44100Hz 双声道 → ⑥ ALSA aplay 播放 |
| **异常** | DashScope 失败 → 自动回退 espeak-ng 离线 TTS；USB 音箱未连接 → aplay 无声 |

### 2.11 录音与音频采集

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员（API 调用）、小车麦克风 |
| **Trigger** | `POST /api/audio/record/start` |
| **Main Flow** | ① 接收参数 → ② 后台录音线程（sounddevice 16kHz 单声道）→ ③ 返回 record_id → ④ 前端查询状态 → ⑤ `POST .../record/stop` 手动停止 → ⑥ 返回 WAV bytes |
| **End State** | WAV 音频采集完成 |

### 2.12 实时视频流分发

| 维度 | 内容 |
|------|------|
| **Actors** | 操作员（浏览器）、摄像头、Flask MJPEG 服务（6500）、ROS2 rosbridge（8080） |
| **Trigger** | 浏览器访问 `/video_feed` |
| **Main Flow** | ① 深度相机采集 → ② MJPEG 编码（multipart/x-mixed-replace）→ ③ 浏览器 `<img>` 实时渲染 → ④ 同时提供 ROS2 代理 `/snapshot` 端点 |
| **异常** | ROS2 容器未运行 → 静默等待；视频代理离线 → 调度中心返回占位图 |
