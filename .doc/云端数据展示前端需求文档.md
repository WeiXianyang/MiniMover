# 烟火告警云端数据展示 — 前端需求文档

> 目的：给出前端页面设计所需的数据接口、字段含义、UI 建议，供设计师/前端开发参考。
> 后端 API 已就绪，`cloud_platform/server.py` 提供全部读接口。

---

## 一、数据来源

云端 MySQL `dev_fireguard.fire_alarm` 表，字段如下：

| 字段 | 类型 | 说明 | 前端展示建议 |
|------|------|------|-------------|
| `id` | 整数 | 自增主键 | 不展示或作为行 key |
| `event_id` | 字符串 | 车端事件 ID，如 `fire_20260713_143052_123456_001` | 可展示，可复制 |
| `alarm_type` | 枚举 | `confirmed_fire` 确认明火 / `suspected_smoke` 疑似烟雾 / `ai_unavailable` AI 失效 | 用图标+颜色区分 |
| `occurred_at` | ISO8601 | 事件发生时刻（车端时间，带时区） | 表格首要排序字段 |
| `reason` | 字符串 | AI 复核的中文原因，如"画面中部可见明显跳动火焰" | 详情页展示 |
| `confidence` | 0~1 浮点 | AI 置信度 | 进度条或百分比 |
| `evidence_url` | URL | 证据图片（Nginx 静态文件） | 缩略图 + 点击放大 |
| `detection_classes` | 字符串 | 命中类别，如 `fire,smoke` | 标签 |
| `max_confidence` | 0~1 浮点 | 本地 YOLO 检测最大置信度 | 可展示 |
| `local_detection_gone` | 布尔 | 告警时本地目标是否已消失 | 标记 |
| `car_id` | 字符串 | 车辆标识 | 多车筛选用 |
| `received_at` | ISO8601 | 云端入库时刻 | 次要参考 |
| `raw_payload` | JSON | 原始上报完整 JSON | 详情页可折叠展示 |

## 二、API 接口

> 基础路径：`http://8.140.28.233:8000`
> 鉴权：需在请求头带 `Authorization: Bearer <API_TOKEN>`（若云端未开启鉴权则不需要）

### 2.1 告警列表（分页+筛选）

```
GET /api/v1/fire-alarms?page=1&size=20&type=confirmed_fire&car_id=car-01&from=2026-07-01T00:00:00&to=2026-07-31T23:59:59
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `page` | 否 | 页码，默认 1 |
| `size` | 否 | 每页条数，默认 20，最大 200 |
| `type` | 否 | 告警类型：`confirmed_fire` / `suspected_smoke` / `ai_unavailable` |
| `car_id` | 否 | 车辆标识 |
| `from` | 否 | 开始时间（ISO8601） |
| `to` | 否 | 结束时间（ISO8601） |

响应格式：
```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "total": 128,
    "page": 1,
    "size": 20,
    "items": [
      {
        "id": 1,
        "event_id": "fire_20260713_143052_123456_001",
        "alarm_type": "confirmed_fire",
        "occurred_at": "2026-07-13T14:30:52.123456+08:00",
        "reason": "画面中部可见明显跳动火焰",
        "confidence": 0.91,
        "evidence_url": "http://8.140.28.233/evidence/fire_20260713_143052_123456_001_001_initial_143052.jpg",
        "detection_classes": "fire",
        "max_confidence": 0.88,
        "local_detection_gone": false,
        "car_id": "car-01",
        "received_at": "2026-07-13T14:30:55.654321+08:00"
      }
    ]
  }
}
```

### 2.2 告警详情

```
GET /api/v1/fire-alarms/{id}
```

响应比列表多一个 `raw_payload` 字段（完整原始上报 JSON，可折叠展示）。

## 三、建议页面

### 3.1 告警总览页（首页）

| 布局元素 | 说明 |
|----------|------|
| 顶部统计卡片 | 今日告警数、确认火灾数、疑似烟雾数、AI 失效数（各用不同颜色） |
| 筛选栏 | 时间范围选择器、告警类型下拉、车辆下拉，带"查询"按钮 |
| 告警表格 | 按 `occurred_at` 倒序，列：发生时间、类型（图标+文字）、车辆、AI 原因（截断 30 字）、置信度（进度条）、证据图缩略图、操作 |
| 分页 | 底部页码或无限滚动 |

### 3.2 告警详情页

| 布局元素 | 说明 |
|----------|------|
| 返回按钮 | 回到列表 |
| 基本信息卡片 | 事件 ID、告警类型、发生时间、接收时间、车辆 |
| AI 复核结果 | 原因文字、置信度（大号进度条/圆环）、检测命中类别 |
| 证据图片 | 大图展示 `evidence_url`，支持点击放大/下载 |
| 原始数据 | `raw_payload` JSON 可展开折叠区域 |

### 3.3 实时大屏（可选）

| 布局元素 | 说明 |
|----------|------|
| 最近告警滚动列表 | 自动刷新（轮询 API），最新 10 条 |
| 证据图片轮播 | 最近告警的图片自动切换 |
| 统计图表 | 按小时/天的告警趋势柱状图、各类型占比饼图 |
| 设备状态 | 各 `car_id` 最近一次上报时间，超时标红 |

## 四、告警类型视觉建议

| 类型 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| `confirmed_fire` | 🔥 火焰 | 红色 `#E74C3C` | 确认明火，最高优先级 |
| `suspected_smoke` | 💨 烟雾 | 橙色 `#F39C12` | 疑似烟雾，需关注 |
| `ai_unavailable` | ⚠️ 警告 | 灰色 `#95A5A6` | AI 服务失效，系统告警 |

## 五、交互细节要求

1. **时间格式化**：`occurred_at` 是 ISO8601 带时区，前端需转为当地时间显示（如 `2026-07-13 14:30:52`）。

2. **置信度展示**：
   - 列表：小号进度条（如 `████████░░ 91%`）
   - 详情：大号圆环或仪表盘样式
   - `confidence` 可能为 `null`（`ai_unavailable` 告警），此时显示"无"或"N/A"

3. **证据图片**：
   - 列表页：60x60 缩略图，点击弹窗放大
   - 详情页：600px 宽自适应大图，点击新窗口打开原图
   - `evidence_url` 可能为空（图片上传失败或文件已清理），显示占位图"暂无证据"

4. **筛选交互**：
   - 时间范围默认"最近 7 天"
   - 切换筛选项后自动查询
   - 筛选条件体现在 URL query 上，可复制分享

5. **实时刷新**：
   - 列表页每 30 秒自动刷新（保持当前筛选条件）
   - 有新告警时顶部统计卡片数字跳动动画
   - 可手动关闭自动刷新

6. **空状态**：无告警数据时显示引导性空状态页面，不是空白表格。

7. **响应式**：列表在手机端可折叠为卡片式布局。

## 六、接口调用示例（JavaScript）

```javascript
const BASE = 'http://8.140.28.233:8000';
const TOKEN = '<API_TOKEN>';  // 鉴权关闭时可为空字符串

async function fetchAlarms(params = {}) {
  const query = new URLSearchParams({ page: 1, size: 20, ...params });
  const headers = {};
  if (TOKEN) headers['Authorization'] = `Bearer ${TOKEN}`;
  const res = await fetch(`${BASE}/api/v1/fire-alarms?${query}`, { headers });
  const json = await res.json();
  if (json.code !== 0) throw new Error(json.msg);
  return json.data;
}

async function fetchAlarmDetail(id) {
  const headers = {};
  if (TOKEN) headers['Authorization'] = `Bearer ${TOKEN}`;
  const res = await fetch(`${BASE}/api/v1/fire-alarms/${id}`, { headers });
  const json = await res.json();
  if (json.code !== 0) throw new Error(json.msg);
  return json.data;
}

// 使用示例
const { items, total } = await fetchAlarms({
  type: 'confirmed_fire',
  car_id: 'car-01',
  from: '2026-07-01T00:00:00',
  to: '2026-07-13T23:59:59',
  page: 1,
  size: 20
});
```
