# MiniMover 人脸识别（注册 + 1:N 身份识别）说明文档

> **用途**：供组内做 PPT / 答辩 / 对接讲解。  
> **一句话**：在小车上先「注册录入人脸」，再用「一张现场照片」在整个人脸库里搜出「是谁」，并给出置信度。

---

## 1. 要解决什么问题

| 场景 | 需要做什么 | 本系统对应能力 |
|------|------------|----------------|
| 首次把某人加入系统 | 录多张人脸 + 绑定账号 | **注册** |
| 现场不知道是谁 | 拍一张脸，和库里所有人比对 | **1:N 识别（一对多）** |
| 已知账号再校验是不是本人 | 密码 + 可选人脸 1:1 | **登录校验**（可选能力） |

PPT 主线建议突出前两项：**注册入库** + **1:N 认出是谁**。

### 1.1 「1:N」是什么意思

- **1**：现场拍到的 **一张人脸**  
- **N**：人脸库里已注册的 **N 个用户**  
- 系统在库中搜索最相似的人，返回：
  - **身份**（用户名 / 显示名）
  - **相似度 score（0~100）** 以及归一化 **confidence（0~1）**

不是「两张图比一下像不像」（那是 1:1），而是「这张脸在整个库里最像谁」。

## 2. 总体架构（逻辑图，可贴 PPT）

```
┌─────────────┐     HTTP :5000      ┌──────────────────────────────┐
│ 客户端 App  │ ──────────────────► │  小车 FireGuard API           │
│ / 演示网页  │   上传人脸图片       │  api_server.py + face/ 模块    │
└─────────────┘                     └───────────┬──────────────────┘
                                                │
                    ┌───────────────────────────┼───────────────────────────┐
                    ▼                           ▼                           ▼
           ┌────────────────┐        ┌────────────────────┐       ┌─────────────────┐
           │ 本地 SQLite    │        │ 本地人脸照片目录    │       │ 百度云人脸库     │
           │ 用户账号信息   │        │ media/face_images/  │       │ group:user_faces │
           │ face_users.db  │        │ （留底 / 管理）     │       │ （1:N 特征检索） │
           └────────────────┘        └────────────────────┘       └─────────────────┘
```

### 2.1 数据存在哪里（重要结论）

| 数据 | 存在哪 | 干什么用 |
|------|--------|----------|
| 用户名、密码哈希、邮箱、手机号 | **小车本地** SQLite（`MiniMover/face/face_users.db`） | 账号管理、登录 |
| 注册时拍的人脸照片文件 | **小车本地** `人脸识别/media/face_images/<用户名>/` | 留底备份、运维查看 |
| 人脸特征向量 / 检索索引 | **百度云** 人脸库，分组名 `user_faces` | **真正做 1:N 搜索** |

因此：

- **账号和照片在小车上**  
- **「认人」依赖百度云在线能力**（需小车能访问外网）  
- 没有做完全离线本地识别引擎

### 2.2 和整车其它模块的关系

人脸 **挂在同一端口 `:5000`**，与控制面板、导航 API 同一进程：

| 能力 | 地址 |
|------|------|
| 控制面板 | `http://<车IP>:5000/` |
| 导航巡逻 | `http://<车IP>:5000/nav/patrol` |
| **人脸网页** | `http://<车IP>:5000/face` |
| **1:N 接口** | `POST http://<车IP>:5000/api/face/recognize` |

车端一键启动：

```bash
bash ~/MiniMover/scripts/start_all.sh
```

## 3. 核心业务流程

### 3.1 注册流程（入库）

**目标**：把一个新人写进「本地账号」+「百度人脸库」，之后才能被 1:N 认出。

```
用户打开 /face → 切到「注册」
        │
        ├─ 填写：用户名 / 密码 / 邮箱 / 手机号
        ├─ 采集人脸：至少 3 张（不同角度更稳）
        │     · 推荐：打开「小车摄像头」拍照
        │     · 也可：上传本地图片
        ▼
POST /api/face/register  (multipart)
        │
        ├─① 本地创建用户（SQLite）
        ├─② 每张图调用百度 user/add → 写入 group `user_faces`
        ├─③ 成功则本地保存照片文件 + face_token
        └─④ 若百度一张都没成功 → 回滚删除本地用户，返回失败
```

**规则要点（PPT 可单列「规则」页）：**

1. 用户名不可重复  
2. 至少 **3 张** 人脸图  
3. 以百度同步成功为准；全失败则注册失败  
4. 默认相似度阈值后续识别用 **80 分**（百度 score）

### 3.2 1:N 识别流程（认出是谁）★ 主演示路径

**目标**：给一张现场人脸，返回「你是谁」+ 置信度。

```
现场拍一张脸 / 客户端上传一张 JPEG
        ▼
POST /api/face/recognize   字段名：image（或 face_image）
        ▼
小车 → 百度 Face Search（在 group user_faces 内 1:N 搜索）
        ▼
取相似度最高的用户（top1）
        │
        ├─ score < 80  → 失败：「相似度不足，无法确认身份」
        ├─ 库中无人脸  → 失败：「未找到匹配用户」
        └─ score ≥ 80  → 成功
                            identity = 用户名（优先本地库解析显示名）
                            score / confidence 一并返回
```

**成功时客户端应展示：**

- `identity`：识别出的人  
- `score`：0~100（相似度）  
- `confidence`：0~1（= score/100）

### 3.3 登录校验（可选，PPT 可略写）

- 校验本地用户名密码  
- 若附带人脸图：再做人脸 **1:1**（与该用户已存 face_token 比对）  
- 仍用阈值 80  

接口：`POST /api/face/login`

## 4. 怎么用（操作手册）

### 4.1 启动服务（车端）

```bash
# 推荐：控制 + 视频 + 导航接口 + 人脸 一起起
bash ~/MiniMover/scripts/start_all.sh
```

启动后记住车 IP（示例，以实际 `hostname -I` 为准）：

- 常见 USB 共享：`192.168.137.23`  
- 或局域网：`10.227.111.x`

### 4.2 网页注册 / 演示识别（运维 & 现场 demo）

浏览器打开：

```
http://<车IP>:5000/face
```

页面三个 Tab：

| Tab | 用途 |
|-----|------|
| **身份识别** | 1:N：打开摄像头 →「识别此人」→ 看姓名和分数 |
| **登录** | 账号密码（+ 可选人脸） |
| **注册** | 录入新用户（≥3 张脸） |

**采集方式建议：**

1. 点 **「打开小车摄像头」**（走小车 Astra / `:8080`，适合电脑用 HTTP 访问车 IP；本机摄像头在局域网 HTTP 下常被浏览器禁用）  
2. 或 **上传图片**  
3. 「打开本机摄像头」仅在 `localhost` / HTTPS 下可靠  

**注册操作步骤：**

1. 切到「注册」  
2. 填用户名、密码、邮箱、手机号  
3. 打开小车摄像头，连续拍 **3 张**（正面 + 微侧更好）  
4. 提交 → 成功即表示本地+百度都已入库  

**识别操作步骤：**

1. 切到「身份识别」  
2. 打开摄像头对准人脸 →「识别此人」  
3. 查看 `你是：xxx` 与相似度分数  

### 4.3 客户端 / App 对接（软件同事）

只接一个接口即可完成「认出是谁」：

```
POST http://<车IP>:5000/api/face/recognize
Content-Type: multipart/form-data
字段：image=<JPEG/PNG 文件>
```

成功示例：

```json
{
  "ok": true,
  "msg": "识别成功",
  "identity": "zhangsan",
  "score": 95.2,
  "confidence": 0.952,
  "user": {
    "id": "12",
    "username": "zhangsan",
    "email": "a@b.com",
    "score": 95.2,
    "confidence": 0.952,
    "source": "local_db"
  }
}
```

失败时看 `ok: false` 与 `msg`（未匹配 / 分太低 / 服务异常）。

curl 自测：

```bash
curl -X POST "http://<车IP>:5000/api/face/recognize" -F "image=@photo.jpg"
```

更细的字段说明见同目录：  
`人脸识别API接口文档（客户端对接）.md`

可选注册接口（也可用网页代替）：

```
POST /api/face/register
username, password, email, phone + face_images×≥3
```

## 5. 技术实现要点（可作 PPT「技术路线」页）

| 层级 | 技术 |
|------|------|
| 车端 Web/API | Flask，蓝图 `face/`，挂在 `api_server.py`（`:5000`） |
| 本地存储 | SQLite + 文件系统照片 |
| 云端识别 | 百度智能云人脸识别：`user/add`（注册）、`search`（1:N）、`match`（1:1） |
| 人脸分组 | `user_faces` |
| 默认阈值 | score ≥ **80** 判为同一人 |
| 网页 UI | `face/page.html`（注册 / 登录 / 识别） |
| 相机 | 小车 ROS `web_video_server`（`:8080`）；网页可直连 stream，也可走 API 抓帧 |

代码位置（方便开发翻）：

```
MiniMover/
  api_server.py          # 主服务，注册 face 路由
  face/
    routes.py            # /face 页面 + /api/face/*
    baidu.py             # 百度 token、add、search、match
    store.py             # SQLite / 本地图片
    page.html            # 注册识别网页
  scripts/start_all.sh   # 一键启动（含人脸）
```

## 6. 演示话术建议（PPT 讲稿提纲）

1. **背景**：消防 / 安防小车需要知道「面前是谁」，支持白名单人员确认。  
2. **方案**：车端统一 API + 百度云 1:N；本地管账号与照片，云端管特征检索。  
3. **入库**：网页或接口注册，≥3 张脸同步到百度人脸库。  
4. **识别**：客户端拍照上传 → 返回身份与置信度 → 业务侧按阈值处置。  
5. **集成**：与遥控、导航同在 `:5000`，一键 `start_all.sh` 即可演示。  
6. **边界**：需联网访问百度；非 HTTPS 电脑端优先用小车摄像头或上传图。

## 7. 常见问题（答辩 / 联调）

| 问题 | 原因 / 处理 |
|------|-------------|
| 本机摄像头打不开 | 用 `http://车IP` 访问时浏览器禁用 getUserMedia → 用「小车摄像头」或上传图 |
| 识别一直失败 | 是否已注册成功；脸是否清晰/光线；score 是否 &lt; 80 |
| 注册失败 | 百度同步失败（网络/配额）或不足 3 张图 |
| 控制页有遥控、视频没有 | 相机流在 `:8080`，与人脸 API 独立；先确认 `http://车IP:8080/stream?topic=/camera/color/image_raw` |
| Django `:8000` 旧人脸服务 | 遗留栈；**现网以 `:5000` Flask 为准**，勿混用两套库 |

## 8. 一页纸总结（可直接做成 PPT 最后一页）

- **能力**：人脸注册 + **1:N 身份识别**（是谁 + 置信度）  
- **部署**：小车 `start_all.sh` → 服务在 **`:5000`**  
- **使用**：网页 ` /face ` 录入；客户端调 `POST /api/face/recognize`  
- **存储**：账号/照片在**车端本地**；特征检索在**百度云** `user_faces`  
- **判定**：百度 score 默认阈值 **80**
