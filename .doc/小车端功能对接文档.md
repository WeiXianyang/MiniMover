# 小车端功能对接文档

> **目的**：整理小车端（Jetson）目前已实现的所有对外功能、接口、指令，供新模块开发对接参考。
> **范围**：小车端所有功能（不含云平台 `cloud_platform/`，云平台接口见 `.doc/云端API接口文档（前端对接）.md`）。

---

## 一、通信架构总览

小车（Jetson Orin Nano）对外暴露以下通信通道：

| 端口 | 协议 | 用途 | 延迟 | 依赖 |
|------|------|------|:--:|------|
| **5000** | HTTP REST (Flask) | 状态查询、视频代理、音频、地图导航、巡逻控制 | 中 | `api_server.py` + `navigation/` |
| **5001** | TCP 纯文本 | 低延迟底盘控制（绕过 HTTP 栈） | 低 | `api_server.py`（同上进程） |
| **6000** | TCP 自定义帧 | 旧版遥控协议（OpenHarmony APP 使用） | 低 | `rosmaster_main_ori.py` |
| **8080** | HTTP MJPEG | ROS 2 相机视频流 | 低 | Docker `web_video_server` |
| **6500** | HTTP MJPEG | 旧版 Web 视频服务 | 中 | `app.py`（已不推荐） |
| **5900** | VNC | 远程桌面 | 低 | Docker 容器 |

```
                   ┌─────────────────────────────────────────┐
                   │          小车 (Jetson Orin Nano)          │
                   │                                          │
  OpenHarmony ────▶│ TCP:6000  自定义帧协议（摇杆/按钮/四轮） │
  APP              │ TCP:5001  纯文本控制（低延迟旁路通道）    │
                   │                                          │
                   │ HTTP:5000  REST API（状态/视频音频等）   │
  调度中心/PC ◀───▶│                                          │
  语音助手/APP     │ HTTP:8080  MJPEG 视频流（ROS 相机话题）  │
                   │                                          │
  云平台 ◀────────▶│ fire_monitor → CloudAlarmService         │
                   │            → 告警上报 → cloud_platform   │
                   └──────────────────────────────────────────┘
```

小车硬件资源：

| 设备 | 路径 | 说明 |
|------|------|------|
| 底盘串口 | `/dev/myserial` | MCU 通信（UART） |
| 激光雷达 | `/dev/rplidar` | RPLIDAR A1（12m 范围） |
| 深度相机 | `/dev/astradepth` | Astra Pro Plus |
| 彩色相机 UVC | `/dev/astrauvc` | 相机 RGB 流 |
| 传感器板 | `/dev/ttyUSB2` | 温湿度/烟雾/PM2.5/气压/光照/CO2/GPS |
| 麦克风 | `hw:2,0` | 讯飞 XFM-DP（mono 16kHz） |
| 扬声器 | C-Media USB | PulseAudio（立体声 44.1kHz） |

## 二、HTTP REST API（端口 5000）

> 实现文件：`api_server.py`（基于 Flask + Flask-CORS）
> 所有响应格式：`{"code": 0, "msg": "ok", "data": {...}}`
> `code`: 0 = 成功，其他 = 错误

### 2.1 系统状态

#### `GET /api/health` — 健康检查

```
响应：{"code": 0, "msg": "FireGuard API Running"}
```

#### `GET /api/status` — 全状态快照

返回小车完整状态：传感器数据 + 电池电压 + 本机 IP。

```json
{
  "code": 0,
  "data": {
    "sensors": {
      "temperature": 25.0,
      "humidity": 50.0,
      "smoke": 0,
      "pm25": 0,
      "pressure": 1013.0,
      "light": 500,
      "co2": 400,
      "gps": { "lat": 0, "lon": 0 }
    },
    "battery": 12.0,
    "ip": "192.168.137.23"
  }
}
```

#### `GET /api/sensors` — 仅传感器数据

返回与 `/api/status` 中 `sensors` 字段相同的结构。

### 2.2 底盘控制

#### `POST /api/move` — 控制移动

```json
// 请求体
{
  "cmd": "forward",    // forward / backward / left / right / left_shift / right_shift / stop
  "speed": 50,         // 速度百分比 10~100
  "duration": 0.5      // 持续时间（秒），stop 时忽略
}
```

```json
// 响应
{"code": 0, "msg": "forward @ 50%"}
```

| cmd 值 | 动作 | 说明 |
|--------|------|------|
| `forward` | 前进 | |
| `backward` | 后退 | |
| `left` | 左转 | |
| `right` | 右转 | |
| `left_shift` | 左平移 | 麦克纳姆轮特有 |
| `right_shift` | 右平移 | 麦克纳姆轮特有 |
| `stop` | 紧急停止 | 忽略 speed/duration |

> **注意**：该接口有串口写锁 + 自动停止定时器，防止死锁和失控。`stop` 命令会取消所有进行中的定时移动。

### 2.4 地图导航

> 实现目录：`navigation/`（Flask Blueprint，挂载于 `/api/nav`）
> 架构：纯 Python 服务通过 `docker exec` 代理所有 ROS 2 调用，无需在 API Server 安装 ROS

**导航系统架构：**

```
HTTP 请求 → Flask Blueprint → ros_bridge.py → docker exec → ROS2 (Nav2/GMapping/AMCL)
                                  ↑                              │
                                  └── stack_manager.py ←── 容器生命周期管理
```

**关键设计：**
- **Docker 代理**：所有 ROS 2 调用（服务/话题/动作）通过 `docker exec <容器> bash -c "source ros_setup && ros2 ..."` 执行
- **超时缓存**：`container_running()` 和 `patrol_services_ready()` 有 2 秒缓存 TTL，防止 Docker exec 风暴
- **异步生命周期**：启动/停止导航栈在后台线程执行，HTTP 请求立即返回
- **初始位姿可靠性**：`/initialpose` 通过 `subprocess.Popen` 非阻塞发布 5 次（含 15s 超时），提高 AMCL 接收概率

#### 2.4.1 地图查询

##### `GET /api/nav/map` — 地图元数据

```json
{
  "code": 0,
  "data": {
    "map": "loudao",
    "width": 384, "height": 384,
    "resolution": 0.05,
    "origin": [-10.0, -10.0, 0.0]
  }
}
```

- 可选参数：`?map=<name>`（默认从 `NAV_MAP_NAME` 环境变量读取，默认 `loudao`）

##### `GET /api/nav/map/image` — 地图图片

返回 `image/jpeg` 格式的地图图像。从 Docker 容器内 PGM 文件（支持 P2 ASCII / P5 二进制）读取，转换为彩色 JPEG（白色=空闲，灰色=未知，深灰=障碍物）。

##### `POST /api/nav/coord/pixel_to_map` — 坐标转换

将屏幕/像素坐标转换为地图世界坐标。

```json
// 请求体（屏幕坐标模式）
{
  "screen_x": 200, "screen_y": 150,
  "display_width": 800, "display_height": 600,
  "map": "loudao"
}
// 或像素坐标模式
{
  "pixel_x": 100, "pixel_y": 200,
  "map": "loudao"
}
```

```json
// 响应
{
  "code": 0,
  "data": { "x": -5.0, "y": -2.5, "pixel_x": 100, "pixel_y": 200 }
}
```

- 转换公式：`x = px * resolution + origin[0]`，`y = (height - py) * resolution + origin[1]`

#### 2.4.2 导航栈生命周期

##### `GET /api/nav/stack/status` — 导航栈状态

```json
{
  "code": 0,
  "data": {
    "container": "ded7",
    "container_running": true,
    "patrol_ready": true,
    "stack_ready": true,
    "starting": false,
    "stopping": false,
    "ros_domain_id": "30",
    "stack_message": "导航栈已就绪",
    "log_tail": "...",
    "hint": null
  }
}
```

- `hint` 字段在栈未就绪时提供操作指引（如 `"点「启动导航栈」自动启动容器"`）

##### `POST /api/nav/stack/start` — 启动导航栈

```json
// 请求体（可选）
{ "wait_ready": false, "timeout": 45 }
```

```json
// 响应
{
  "code": 0,
  "msg": "已开始启动，请等待状态变为「就绪」（约 15 秒）",
  "data": { "accepted": true, "starting": true, "patrol_ready": false }
}
```

- 异步启动：立即返回，后台线程执行 `docker start` + 栈启动脚本
- `wait_ready=true` 时阻塞等待直到就绪或超时
- 启动流程：启动容器 → 执行 `nav_stack.sh start` → 轮询等待巡逻服务注册（最多 ~50s）

##### `POST /api/nav/stack/stop` — 关闭导航栈

```json
{
  "code": 0,
  "msg": "正在关闭导航栈…",
  "data": { "accepted": true, "patrol_ready": false }
}
```

#### 2.4.3 初始位姿

##### `POST /api/nav/initial_pose` — 设置 AMCL 初始位姿

```json
// 请求体
{ "x": 0.0, "y": 0.0, "yaw": 0.0 }
```

```json
// 响应
{
  "code": 0,
  "msg": "起点已提交 (0.00, 0.00, yaw=0.00)",
  "data": { "success": true, "x": 0.0, "y": 0.0, "yaw": 0.0 }
}
```

- 通过 `ros2 topic pub --times 5 /initialpose` 发布（含四元数转换 `z=sin(yaw/2), w=cos(yaw/2)`）
- 非阻塞 fire-and-forget，不等待 DDS 发现

```json

```

```json
    "loop": true
  }
}
```

```json
// 请求体
{
  "points": [ { "x": 1.0, "y": 2.0 } ],
  "map": "loudao",
```

```json

```
```json
// 响应
{
  "code": 0,
  "msg": "路线已加载",
  "data": { "route": { ... }, "loaded": { "success": true } }
}
```

- 自动调用 `set_initial_pose` + `set_route`
- `apply_initial_pose: false` 可跳过初始位姿设置

### 2.8 TCP 直连控制通道（端口 5001）

> 实现：`api_server.py` 内置，与 HTTP :5000 共用同一进程和 `bot` 实例。

**设计目的**：HTTP `/api/move` 每次请求都经过 TCP 三次握手 + Flask 路由 + JSON 解析，延迟 10-50ms+。`:5001` 是纯 TCP socket，旁路 HTTP 栈，直接用换行符分隔的纯文本指令执行 `_execute_move()`，延迟降至 ~2-5ms（与旧版 :6000 同水平，但没有 hex 帧协议的开销）。

**:5000 和 :5001 差异对比：**

| 维度 | HTTP :5000 `/api/move` | TCP :5001 |
|------|------------------------|-----------|
| 协议 | HTTP POST + JSON body | 纯文本换行分隔 |
| 建连 | 每次请求 TCP 三次握手 | 长连接，一次建连 |
| 解析 | Flask 路由 + `request.json` 反序列化 | `split()` 直接解析 |
| 锁 | 共享 `_move_lock`（与 :5001 互斥） | 共享 `_move_lock`（与 :5000 互斥） |
| 延迟 | ~10-50ms+ | ~2-5ms |
| 返回 | `{"code":0,"msg":"forward @ 50%"}` | 无返回（fire-and-forget） |

**协议格式**（每行一条指令，换行符 `\n` 分隔）：

```
<cmd> [speed_pct] [duration_sec]
```

| 字段 | 说明 | 默认值 |
|------|------|:---:|
| `cmd` | 必需，同 HTTP `cmd` 值（forward/backward/left/right/rotate_left/rotate_right/left_shift/right_shift/stop） | — |
| `speed_pct` | 速度百分比 10~100 | 50 |
| `duration_sec` | 持续时间秒数 | 0.5 |

**示例：**

```
forward 50 0.3
stop
left 30 0.5
rotate_right 60 1.0
```

**Python 客户端调用：**

```python
import socket

def tcp_move(car_ip, cmd, speed=50, duration=0.5):
    """通过低延迟 TCP :5001 控制小车"""
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.settimeout(2)
    try:
        s.connect((car_ip, 5001))
        s.sendall(f"{cmd} {speed} {duration}\n".encode())
    finally:
        s.close()

# 使用
tcp_move("192.168.137.23", "forward", speed=50, duration=0.5)
tcp_move("192.168.137.23", "stop")
```

**性能优化提示：** 连续发送多条指令时，保持一条长连接比频繁建连更省延迟：

```python
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.connect(("192.168.137.23", 5001))
s.sendall(b"forward 50 0.3\n")   # 第 1 条 ~2ms
s.sendall(b"left 30 0.2\n")      # 第 2 条 ~2ms (无握手开销)
s.sendall(b"stop\n")
s.close()
```

## 三、传统 TCP 控制协议（端口 6000）

> 实现文件：`rosmaster_main_ori.py`
> 使用方：OpenHarmony 遥控 APP
> 协议说明：自定义 ASCII-Hex 文本帧，基于 TCP 长连接

### 3.1 帧格式

```
$ + [车辆类型 2字符] + [命令标记 2字符] + [数据长度 2字符] + [数据主体 变长] + [校验和 2字符] + #
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 起始符 `$` | 1 | 固定 |
| 车辆类型 | 2 | 固定 `01` |
| 命令标记 | 2 | 见下表 |

### 3.3 方向枚举（命令 15）

| 值 | 含义 | 值 | 含义 |
|:--:|------|:--:|------|
| 0 | 停车 Stop | 4 | 右平移 Right |
| 1 | 前进 Front | 5 | 左旋转 LeftRotate |
| 2 | 后退 After | 6 | 右旋转 RightRotate |
| 3 | 左平移 Left | 7 | 刹车 Brake |

### 3.4 编码示例

「前进」按钮（方向 = 1）：

```
1. 数据主体 = '01'，长度 = 2
2. 数据长度字段 = (2 + 2) 的 16 进制 → '04'
3. 拼接 = '01' + '15' + '04' + '01' = '01150401'
4. 校验和 = 0x01 + 0x15 + 0x04 + 0x01 = 0x1B
5. 追加校验 → '011504011B'
6. 最终帧: $011504011B#
```

### 3.5 连接行为

- **无心跳保活**：5 秒定时器内有占位代码，但心跳发送逻辑已被注释。
- **惰性重连**：每次 `sendMessage()` 前检测 `isConnect()`，若断开则重连（重连当帧数据不补发）。
- **单向通信**：当前 APP 只发不收，接收处理逻辑已被注释。

## 七、语音控制服务

> 实现目录：`voice_assistant/`
> 服务入口：`voice_assistant/voice_service.py`

### 7.1 架构

```
麦克风 → VAD → FunASR/Whisper → 唤醒词门控 → 指令解析 → /api/move → (可选) LLM → (可选) TTS 播报
                ↑                ↑                            │
                │                └── WakeWordVoiceService      │
                │                     IDLE ←→ AWAKE (30s)      │
                └── 声纹验证（CAM++ 余弦相似度 ≥0.38）          │
                                                               ↓
                                                    扬声器 /api/audio/say
```

### 7.2 三种 ASR 后端

| 后端 | 文件 | 模式 | 说明 |
|------|------|------|------|
| **FunASR** | `asr_backend.py::FunAsrBackend` | 本地 | Paraformer 2-pass（在线流式 + 离线确认），+ silero-vad |
| **Whisper** | `asr_backend.py::WhisperBackend` | 远程 HTTP | 本地 VAD + Whisper API 转录 |

### 7.3 支持指令

| 指令 | 触发词（中文） | 动作 |
|------|---------------|------|
| `stop` | 急停 / 停止 / 停下来 / 停下 / 别动 / 不要动 | 紧急停止（绕过声纹验证） |
| `spin` | 转圈 / 原地转 / 旋转一圈 | 原地旋转 |
| `backward` | 向后转 / 后转 / 向后走 / 后退 | 后退 |
| `left` | 向左转 / 左转 / 往左转 | 左转 |
| `right` | 向右转 / 右转 / 往右转 | 右转 |

### 7.4 声纹验证

- **模型**：CAM++（FunASR）
- **阈值**：余弦相似度 ≥ 0.38
- **规则**：
  - 运动指令（前进/后退/转弯）= 需要声纹验证通过
  - `stop` 指令 = 始终放行（安全优先级）
- **注册**：需要预先录入说话人声纹样本

### 7.5 唤醒词服务（WakeWordVoiceService）

> 实现文件：`voice_assistant/wake_word.py`（main 分支提交 `6a26a53`）
> 核心类：`WakeWordVoiceService`

`WakeWordVoiceService` 是对 `VoiceService` 的包装器，在原有语音管道之上增加了唤醒词门控。

**状态机：**

```
IDLE（静默监听）
   │
   │ 检测到唤醒词（如"你好小南"）
   ▼
AWAKE（指令监听）
   │
   │ 收到语音指令 → 正常处理（走原有 VoiceService 管道）
   │ 30 秒无语音 → 超时回退
   │ 收到 stop 指令 → 回退
   ▼
IDLE（静默监听）
```

**关键参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `wake_word` | `""`（必须配置） | 唤醒词，如 `"你好小南"` 或 `"小北"` |
| `greeting` | `"你好，我是{wake_word}，有什么可以帮你的？"` | 唤醒后的 TTS 欢迎语 |
| `idle_timeout` | `30.0` 秒 | 唤醒后无语音交互自动回退 IDLE 的时间 |

**工作流程：**

1. **IDLE 状态**：持续监听每个 final utterance，扫描其中是否包含 `wake_word`
2. **检测到唤醒词**：`self._awake = True`，记录唤醒时间，通过 TTS 播报欢迎语
3. **AWAKE 状态**：后续语音文本直接交给原有 `VoiceService.handle_final()` 处理（含指令解析、声纹验证、小车控制等）
4. **超时回退**：若 `idle_timeout` 秒内无新语音输入，自动回到 IDLE
5. **stop 指令**：收到 stop 后回到 IDLE

**启动方式：**

```python
from voice_assistant.wake_word import WakeWordVoiceService

wake_svc = WakeWordVoiceService(
    voice_service=voice_svc,
    wake_word="你好小南",
    greeting="小南已就绪，请下达指令",
    idle_timeout=30.0,
)
wake_svc.run()  # 阻塞运行，Ctrl+C 中断
```

### 7.6 模型加载完成通知（on_ready）

> main 分支提交 `1180274`

`FunAsrBackend` 的构造函数新增 `on_ready` 回调参数：

```python
class FunAsrBackend(AsrBackend):
    def __init__(self, sample_rate=..., silence_ms=..., vad_threshold=0.35, on_ready=None):
        self._on_ready = on_ready
```

在 ASR 模型加载完成（离线模型初化 + 并行在线模型 + silero-vad ready）后，会调用 `self._on_ready()`，通知调用方模型已就绪。

**用途**：启动脚本或上层调度器可以等待 ASR ready 后再执行后续操作（如播放提示音），避免在模型还在加载时用户就开始说话。

| `/redoc/` | GET | ReDoc API 文档 |

### 8.2 注册流程

```
1. POST /api/register
   multipart/form-data:
     username, password, email, phone
     face_images: ≥3 张人脸照片

2. 服务端：
   → 密码 AES-ECB 加密存储
   → 邮箱/手机 AES-ECB 加密存储
   → 人脸照片 AES-EAX 加密存储到磁盘
   → 人脸同步注册到百度云人脸库（获取 face_token）

3. 返回：user info
```

### 8.3 1:N 识别

```
POST /api/face_recognition
multipart/form-data: image (单张人脸照片)

响应：
{
  "identity": "matched",
  "user": { "id": 1, "username": "admin", "email": "a***@example.com", "score": 95.2 },
  "candidates": [
    { "user_id": 1, "user_name": "admin", "score": 95.2 },
    { "user_id": 2, "user_name": "inspector1", "score": 42.1 }
  ]
}
```

- 识别阈值：百度云 `FACE_MATCH_THRESHOLD = 80`
- 使用百度云 Face Search API

### 8.4 对接小车视频流

```bash
# 从小车拉取视频流做人脸识别
python identify_car_stream.py --car-ip 192.168.137.23
```

从 `http://<car_ip>:8080/stream?topic=/camera/color/image_raw` 拉流，逐帧检测人脸并调用识别 API。

### 8.5 安全说明

- 密码、邮箱、手机号：AES-ECB 加密后存入数据库
- 人脸图片文件：AES-EAX 加密后存磁盘
- 百度云 token：仅存储用于人脸匹配的 face_token

### 9.2 告警类型

| AI 超时 | 30 秒 | 单次 AI 复核请求超时 |
| AI 最大重试 | 3 次 | 超时后的重试次数 |
| 证据最大数量 | 10 张 | 本地保留证据图片上限 |

## 十二、传感器板

> 实现文件：`sensors/icar_sensor_driver.py`
> 硬件接口：`/dev/ttyUSB2`，115200 baud

### 数据帧格式

6 字节帧：`[0xA5][节点ID][传感器类型][数据高字节][数据低字节][校验和]`

### 传感器类型与换算

| 传感器 | 换算公式 | 单位 |
|--------|----------|------|
| 温度 | value / 10.0 | °C |
| 湿度 | value / 10.0 | % |
| 烟雾 | raw value | — |
| PM2.5 | raw value | μg/m³ |
| 气压 | value / 10.0 | hPa |
| 光照 | raw value | lux |
| CO2 | raw value | ppm |
| GPS 纬度 | value / 1000000.0 | 度 |
| GPS 经度 | value / 1000000.0 | 度 |

### 读取方式

- 通过小车 HTTP API：`GET /api/sensors` 或 `GET /api/status`
- 直接 Python 调用：

```python
from sensors.icar_sensor_driver import iCarSensorDriver

driver = iCarSensorDriver()
data = driver.get_sensor_data()
# {"temperature": 25.0, "humidity": 50.0, "smoke": 0, ...}
```

> 注意：传感器驱动有线程锁（`threading.Lock`），多线程安全。

## 十三、底盘控制底层

> 实现文件：`Rosmaster_Lib/`（Cython 编译，部分为 `.so`）
> 通信方式：UART 串口 `/dev/myserial`

### 支持底盘类型

| 类型 | 名称 | 驱动方式 |
|------|------|----------|
| X3 | 标准麦克纳姆轮底盘 | 四轮独立 |
| X3 Plus | 带机械臂麦克纳姆轮底盘 | 四轮独立 + 机械臂 |
| X1 | 两轮差速底盘 | 差速驱动 |
| R2 | 阿克曼转向底盘 | 前轮转向 + 后轮驱动 |

### 速度范围

| 底盘类型 | X 线速度 | Y 平移速度 | Z 角速度 |
|----------|----------|------------|----------|
| R2 | 0 ~ 1.8 | 0.045 | 3.0 |
| X3 | 0 ~ 1.0 | 1.0 | 5.0 |
| X3 Plus | 0 ~ 0.7 | 0.7 | 3.2 |

### 直接调用（绕过 HTTP API）

```python
from Rosmaster_Lib import Rosmaster

car = Rosmaster()
car.set_car_motion(0.5, 0, 0)    # 前进 0.5m/s
car.set_car_motion(0, 0, 0)      # 停止
```
