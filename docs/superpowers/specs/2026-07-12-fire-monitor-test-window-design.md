# 火情链路临时可视化窗口设计

## 目标

提供一个仅用于联调的 Tkinter 小窗口，以真实摄像头或本地视频启动现有 YOLO 检测器，并查看实时检测画面、触发进度、证据抓拍、实际发送给 AI 的无框图片、AI 重试/结果及报警状态。

## 方案

窗口以子进程运行现有 `detector.py`，不复制推理逻辑。检测子进程通过 `runtime/debug/` 中原子更新的 `status.json`、`latest_frame.jpg`、`latest_ai_upload.jpg` 和追加式 `events.jsonl` 输出调试遥测。窗口每 200ms 读取这些文件并刷新；停止或关闭窗口时终止子进程并释放摄像头。

## 范围

- 来源支持摄像头编号和本地视频。
- 只调用 `.env`/环境变量配置的真实 AI，不提供模拟结果。
- 调试图片不进入证据图 10 张轮转集合。
- 窗口展示检测命中数、事件状态、AI 尝试次数/耗时/结果、证据路径和报警状态。
- 新增 `--no-view` 避免测试窗口运行时再弹 OpenCV 窗口；新增 `--monitor-debug-dir` 开启遥测。

## 安全与失败处理

调试状态不包含 API Key 或 Authorization。JSON/JPEG 使用临时文件替换，窗口不会读到半写文件。AI 未配置时明确显示失败；AI 失败仍沿用现有三次真实请求及人工介入报警逻辑。
