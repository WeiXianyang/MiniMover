# 三项识别答辩演示设计

## 目标

为 MiniMover 的车牌识别、红绿灯识别、火焰烟雾识别提供一个 Windows 一键演示入口，适配约 5 分钟答辩。演示必须启动快、离线可用、无需摄像头或网络，并在一次双击后同时显示三个独立窗口。

## 用户体验

仓库根目录新增 `一键启动三项识别演示.bat`。双击后执行环境检查并同时启动三个独立播放器进程：

1. 车牌识别演示窗口；
2. 红绿灯识别演示窗口；
3. 火焰烟雾识别演示窗口。

三个窗口循环播放 10–15 秒的演示素材，窗口标题和画面内中文标签明确标识模块。按 `Q` 或 `Esc` 可关闭单个窗口；关闭启动器终端不强制杀死其他窗口。

## 演示素材

新增独立的 `demo_showcase/` 目录，包含：

- 三项演示播放器及启动配置；
- 三段体积受控的模块演示视频；
- 一段三画面合成兜底视频 `三项识别答辩演示.mp4`；
- 使用说明和依赖检查。

素材从仓库内已有识别结果、测试图片和视频中选取或生成，不依赖答辩现场推理。画面只展示可以由现有项目资产证明的识别结果，不虚构精度、速度或现场检测结论。

## 技术方案

使用 Python + OpenCV 播放本地 MP4。统一播放器接收视频路径、窗口标题和屏幕位置参数。BAT 使用三个独立 `start` 命令并行启动播放器，并在缺少 Python、OpenCV或素材时输出明确错误和兜底视频路径。

视频优先使用系统已有 FFmpeg 生成；若 FFmpeg 不可用，则使用 OpenCV `VideoWriter`。编码选择 Windows 常见播放器和 OpenCV 均易读取的 MP4/MPEG-4，控制分辨率、帧率和时长以缩短启动时间。

## 稳定性策略

- 默认离线播放预生成结果，不同时加载三个大型模型；
- 所有路径相对于 BAT 自身目录解析，不写死 `E:\MiniMover`；
- 三个进程相互独立，一个窗口失败不阻塞另外两个；
- 保留合成 MP4 作为无需 Python 的答辩兜底；
- 启动前执行快速自检，验证 Python、OpenCV和四个视频文件。

## 范围边界

- 不修改三个识别模块的核心算法；
- 不把三个模型整合成 Web 页面；
- 不修改或暂存既有未跟踪的 `app_pc.py`、根目录 `run.sh`、`traffic_light/`、`oh-ai-car-ros-app/`；
- 演示素材和启动器放入新的受 Git 管理目录；
- 完成验证后，将当前功能分支合并进 `main`，不自动推送远程。

## 验收标准

1. 双击 BAT 可并行启动三个视频窗口；
2. 每个视频能够打开、读取首帧并循环播放；
3. 合成兜底 MP4 可由系统默认播放器打开；
4. BAT 和 Python 路径不依赖固定盘符；
5. 原有火焰烟雾模块 7 项测试继续通过；
6. Git 状态只保留用户明确要求不触碰的四项既有未跟踪文件；
7. 功能分支无冲突合并到 `main`。
